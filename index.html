Introduction to the Business Case

Customer segmentation is a key challenge in e-commerce, where businesses must group their customers to optimize marketing strategies, product recommendations, and logistics. By identifying customer groups, businesses can target them with personalized offerings, resulting in higher sales, better customer retention, and enhanced user experience.

The problem is to analyze a large dataset of customer purchases and behaviors to:
  • Identify distinct customer groups.
  • Sort customers by their total spending, frequency of purchases, or geographical proximity.
  • Allow efficient searching for specific customers based on various attributes like spending patterns.

Portable Tools

Why Merge Sort?
  • Definition: A stable sorting algorithm preserves the relative order of elements with equal keys
  • Benefit: In customer segmentation, if two customers have the same total spending or purchase frequency, their original order (e.g., based on region or signup date) will remain unchanged. This is crucial for preserving data integrity.
Why Quick Sort?
  • Definition: Quick Sort performs sorting directly in the original array without requiring extra memory for temporary storage (unlike Merge Sort, which requires auxiliary space).
  • Benefit: Quick Sort is memory-efficient, making it a better choice for applications where minimizing space usage is important (e.g., sorting customer data stored in arrays). It’s especially useful for scenarios involving limited memory environments.
Why Binary Search?
  • Definition : Binary Search is an efficient algorithm used to find the position of a target element in a sorted dataset by repeatedly dividing the search range in half.
  • Benefits:It is ideal when you need to quickly retrieve or analyze data from sorted customer datasets. The data might be sorted by metrics like spending, purchase frequency, or customer IDs.

void merge(vector& customers, int left, int mid, int right, bool sortBySpending) {
vector temp;
int i = left, j = mid + 1;
while (i <= mid && j <= right) {
if ((sortBySpending && customers[i].totalSpending <= customers[j].totalSpending) ||
(!sortBySpending && customers[i].name <= customers[j].name)) {
temp.push_back(customers[i++]);
} else {
temp.push_back(customers[j++]);
}
}
while (i <= mid) temp.push_back(customers[i++]);
while (j <= right) temp.push_back(customers[j++]);
for (int k = left; k <= right; k++) {
customers[k] = temp[k - left];
}
}
void mergeSort(vector& customers, int left, int right, bool sortBySpending = true) {
if (left >= right) return;
int mid = left + (right - left) / 2;
mergeSort(customers, left, mid, sortBySpending);
mergeSort(customers, mid + 1, right, sortBySpending);
merge(customers, left, mid, right, sortBySpending);
}
int partition(vector& customers, int low, int high, bool sortByFrequency) {
double pivot = sortByFrequency ? customers[high].purchaseFrequency : customers[high].totalSpending;
int i = low - 1;
for (int j = low; j < high; j++) {
if ((sortByFrequency && customers[j].purchaseFrequency < pivot) ||
(!sortByFrequency && customers[j].totalSpending < pivot)) {
swap(customers[++i], customers[j]);
}
}
swap(customers[i + 1], customers[high]);
return i + 1;
}
void quickSort(vector& customers, int low, int high, bool sortByFrequency = false) {
if (low < high) {
int pi = partition(customers, low, high, sortByFrequency);
quickSort(customers, low, pi - 1, sortByFrequency);
quickSort(customers, pi + 1, high, sortByFrequency);
}
}
int binarySearch(const vector& customers, string name) {
int left = 0, right = customers.size() - 1;
while (left <= right) {
int mid = left + (right - left) / 2;
if (customers[mid].name == name) return mid;
if (customers[mid].name < name) left = mid + 1;
else right = mid - 1;
}
return -1; // Not found
}
